이 페이지는 이전 EmuNAND의 데이터를 새로운 커스텀 펌웨어 SysNAND로 옮기고, 이전 EmuNAND 파티션을 제거하는 방법을 안내하는 부가 섹션입니다. EmuNAND와 RedNAND라는 용어는 [동일한 개념](http://3dbrew.org/wiki/NAND_Redirection)을 다른 방식으로 구현한 것이라는 점을 숙지하십시오.
{: .notice--info}

이 페이지를 진행하면 GBA VC 및 DSiWare 세이브가 사라집니다!
{: .notice--danger}

**이 작업을 하기 위해서는 arm9loaderhax + Luma3DS가 설치되어 있어야 합니다.**
{: .notice--danger}

#### 준비물

* 설치된 EmuNAND
* 최신 버전의 [GodMode9](https://github.com/d0k3/GodMode9/releases/latest)

#### 지시 사항

1. GodMode9 `.zip` 파일에서 `GodMode9.bin` 파일을 SD 카드의 `/luma/payloads/` 폴더에 복사하고 `GodMode9.bin` 파일의 이름을 `up_GodMode9.bin`으로 바꾸십시오.
2. SD 카드를 다시 3DS에 삽입하십시오.
3. (Start) 버튼을 누른 채 기기를 켜 Hourglass9을 여십시오.
4. "EmuNAND Backup/Restore", "EmuNAND Backup" 옵션을 차례로 선택하여 가지고 있는 EmuNAND를 `NANDmin_emu.bin` 파일로 백업하십시오.
5. (B) 버튼을 눌러 메인 메뉴로 돌아가십시오.
6. "SysNAND Backup/Restore", "SysNAND Restore (keep a9lh)" 옵션을 차례로 선택하여 `NANDmin_emu.bin` 파일을 SysNAND에 복원하십시오.
7. (Select) 버튼을 눌러 SD 카드를 빼내십시오.
8. SD 카드를 컴퓨터에 삽입한 뒤 SD 카드의 `/files9/` 폴더에서 `NANDmin_emu.bin` 파일과 `NANDmin_emu.bin.sha` 파일을 안전한 위치에 복사하십시오.
  + 여러 위치에 백업하십시오.
  + 나중에 무언가가 잘못되어 벽돌이 되었을 때 이 백업 파일로 기기를 살릴 수 있습니다.
9. 안전한 위치에 복사한 뒤 SD 카드의 `/files9/` 폴더에서 `NANDmin_emu.bin` 파일을 삭제하십시오.
10. **SD 카드의 모든 파일을 컴퓨터에 백업하십시오. 다음 단계에서 SD 카드의 모든 파일이 삭제됩니다.**
11. SD 카드를 다시 3DS에 삽입하십시오.
12. (Start) 버튼을 누른 뒤 (십자 위) 버튼을 누른 채 기기를 다시 시작하여 GodMode9을 여십시오.
13. 홈 버튼을 눌러 액션 메뉴를 여십시오.
14. "SD format menu"를 선택하십시오.
15. (A) 버튼을 눌러 진행하십시오.
16. "No EmuNAND"를 선택하십시오.
17. "Auto"를 선택하십시오.
18. 주어진 키 조합을 입력하여 진행하십시오.
19. (R) 버튼과 (B) 버튼을 동시에 눌러 SD 카드를 빼내십시오.
20. SD 카드를 컴퓨터에 삽입한 뒤 백업했던 모든 파일을 다시 SD 카드에 복사하십시오.
  + SD 카드의 `arm9loaderhax.bin` 파일을 백업에 포함된 파일로 교체하십시오.
21. SD 카드를 다시 3DS에 삽입하십시오.
22. (A) 버튼을 눌러 SD 카드를 다시 마운트하십시오.
23. (Start) 버튼을 눌러 기기를 다시 시작하십시오.
24. 검은 화면이 보인다면 [이 문제 해결 가이드](troubleshooting#ts_sys_down)를 참고하십시오.
